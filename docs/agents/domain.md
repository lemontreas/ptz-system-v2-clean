# Domain Docs

本仓库采用 single-context 布局。

## Before exploring

- 先读取根目录 `CONTEXT.md`，使用其中定义的领域术语。
- 如存在 `docs/adr/`，再读取与当前工作相关的 ADR。
- 文档缺失时静默继续；只有在术语或重要决策确实形成后，才由相应 producer skill 创建。

## Vocabulary

输出中的领域概念应沿用 `CONTEXT.md` 的 canonical term，避免使用其中明确列入 `_Avoid_` 的近义表达。

## ADR conflicts

如拟议方案与已有 ADR 冲突，应明确指出，不得静默覆盖。
