# 产品特性 - WiFi信号定位设备

> **文档定位**：描述产品能做什么、各功能的边界和特性。以产品 / 前端 / 后端共同认可的内容为准。
> **更新规则**：新增或变更特性时同步更新本文档。

---

## 一、产品概述

本设备是一套 **WiFi 信号方向定位系统**，由 PTZ 云台 + WiFi 网卡 + 摄像头组成。

核心能力分为两大模块：

- **定位**：找到目标 WiFi 设备（按 MAC 地址）在扫描区域内信号最强的方向
- **描绘**：对指定区域内所有 WiFi 设备进行信号分布全面采集，生成可视化全景视图

---

## 二、设备使用前提与约束

> 以下约束由硬件安装决定，用户在使用前需了解。


| 约束            | 说明                                                                                          |
| ------------- | ------------------------------------------------------------------------------------------- |
| **与目标保持有效距离** | 摄像头与天线之间存在固定高度差（物理安装决定）。距离目标过近时，高度差产生的角度偏差不可忽略，会导致图像映射与天线实际指向不准确。设备需在与目标保持**一定有效距离**的前提下使用。 |
| **适用场景**      | 适合对远距离目标进行方向定位，不适合极近距离（如同一房间内极短距离）的精确映射。                                                    |

## 五、定位特性

**核心目标**：找到目标设备信号最强的方向，在全景图上高亮标注。


| 特性       | 说明                                 | 状态               |
| -------- | ---------------------------------- | ---------------- |
| 对角框选区域   | 通过移动云台 + 摄像头画面，确认左上/右下两个对角点来选定扫描范围 | ✅                |
| 单设备定位    | 输入 1 个 MAC 地址，定位该设备信号最强方向          | ✅                |
| 多设备定位    | 输入多个 MAC 地址，同时对多个设备定位              | ✅（展示方式待定，见§六 P1） |
| 信道自动探测   | 信道/带宽为可选项，不填时后端自动遍历全部信道/带宽组合探测     | ✅                |
| 信号最强方向标注 | 算法计算出信号最强点，在全景图上高亮显示               | ✅                |

## 八、后续产品特性候选池

> 记录已有方向但尚未立项的特性，供后续规划参考。

### F1. 双天线信号融合分析

**背景**：设备同时配备定向天线和全向天线，当前已能采集双路 RSSI，但尚未做深度分析。

**方向**：利用定向天线与全向天线的 RSSI 差值，通过算法推断目标设备的多维特性，例如：

- 远距离 / 近距离判断
- 信号是否受遮挡
- 其他可推断的设备位置特征

**思路**：

- 两者 RSSI 差值大 → 目标大概率较远（定向增益明显，全向收不到）
- 两者 RSSI 接近 → 目标较近或在全向覆盖范围内
- 具体算法和判定阈值待实测数据支撑后设计

**状态**：🔲 待规划

---

### F2. 定向天线对准后的扩展操作能力

**背景**：定位功能找到目标设备信号最强方向后，定向天线已对准目标。此时可利用定向天线增益高、链路质量好的优势，叠加更多操作能力。

**方向**：在对准状态下支持挂载扩展操作模块，例如：

- 密码本暴力破解（Handshake 捕获 + 字典攻击）
- 其他后续可扩展的主动操作

**架构思路**：定位完成 → 云台锁定最强方向 → 用户选择触发扩展操作 → 后端调度对应模块执行

**状态**：🔲 待规划
