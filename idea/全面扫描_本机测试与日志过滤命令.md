# 全面扫描 Linux 本机测试与日志过滤命令

> 本文件里的“本机”指 Linux 设备本机。所有命令都在 Linux 设备的项目根目录执行。

## 1\. 推荐：启动后端，正常输出并保存完整日志

```
sudo env PYTHONUNBUFFERED=1 /home/ultiwill/ptz_capture_system/ptz_env/bin/python -u run_manager.py 2>&1 \
  | tee full_area_runtime.log
```

默认 Web 地址：

```
http://127.0.0.1:5000
```

说明：

```
python -u + PYTHONUNBUFFERED=1 = 避免管道缓冲，屏幕实时输出
tee full_area_runtime.log      = 屏幕正常显示全部程序日志，同时保存完整日志
```

测试时推荐用这个命令。运行过程中不用过滤，先保留完整现场；测试完再从日志里过滤全面扫描相关信息。

## 2\. 设置测试范围

测试用小范围：

```
粗筛范围：水平 35~65，垂直 0~20
工作区：  水平 48~52，垂直 4~8
```

写入 Redis：

```
redis-cli HSET gimbal:default_config work_pan_range "[35,65]" work_tilt_range "[0,20]" pan_range "[48,52]" tilt_range "[4,8]"
```

这里故意写成一行，避免复制多行命令时反斜杠后面混入空格，导致 `ERR wrong number of arguments for 'hset' command`。

含义：

```
work_pan_range/work_tilt_range = 全面扫描 precheck 粗筛范围
pan_range/tilt_range           = 前端不传工作区时，后端使用的工作区范围
```

## 3\. 启动全面扫描

单轮扫描（执行一轮后停止）：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/ptz/full_area_scan/start" \
  -H "Content-Type: application/json" \
  -d '{
    "project_name": "linux_full_scan_test",
    "work_ranges": [
      {
        "pan_range": [48, 52],
        "tilt_range": [4, 8]
      }
    ]
  }' | python3 -m json.tool
```

连续扫描（窗口 10 分钟，间隔 60 秒，持续循环直到手动停止）：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/ptz/full_area_scan/start" \
  -H "Content-Type: application/json" \
  -d '{
    "project_name": "linux_full_scan_test",
    "work_ranges": [
      {
        "pan_range": [48, 52],
        "tilt_range": [4, 8]
      }
    ],
    "scan_time_limit": 50,
    "time_interval": 60
  }' | python3 -m json.tool
```

注意：全面扫描各阶段停留时长由 `config.json` 的 `"全面扫描"` 配置决定：

```
粗扫每信道停留时长
细扫每信道停留时长
偏差区每信道停留时长
```

修改 `config.json` 后需要重启后端才会重新读取。

```

运行日志里应能看到当前 worker 的三段执行：

```

阶段三

## 3\. 当前定位扫描逻辑检查点

```
channel_probe: 每个范围中心 + 四角，最多 probe_rounds_max 轮
coarse16:      工作范围外扩 expand_deg（默认10°），固定步径 16°
mid8:          根据上一轮强信号区域收缩，固定步径 8°，单框最多 16 点
fine4:         根据上一轮强信号区域继续收缩，固定步径 4°，单框最多 16 点
locating:      移动到每个 MAC 当前最强点再验证一次
fast_verify:   第二轮起优先回到上一轮 best_pos，固定 4 秒快速校验，通过则立即抓包
```

`probe_dwell_time` 只影响 `channel_probe` 信道探测；`dwell_time` 影响 `coarse16/mid8/fine4/locating` 点位采集，默认 1 秒；`fast_verify` 固定 2 秒。多个 MAC 如果在同一个 `(channel, bandwidth)`，同点位共享一次采集；不同信道组会分别切信道采集。

连续追踪规则：

```
不传 time_interval: 只完整执行一轮
传 time_interval: 每轮完成后等待指定秒数再进入下一轮
不传 track_time_limit: 持续追踪直到手动停止
传 track_time_limit: 到达指定分钟数后自动结束
track_rssi_threshold: 从 config.json「定位扫描.快速校验信号阈值」读取，默认 5.0dB
```

# 一键最强点抓包 (capture\_at\_best) 测试命令

> 前提：已完成至少一轮全面扫描或定位扫描，Redis 中有有效的扫描结果。

## 1\. 基础用法（自动生成 pcap 文件名）

替换 `MAC地址` 为扫描结果中出现的目标 MAC：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/ptz/capture_at_best" \
  -H "Content-Type: application/json" \
  -d '{
    "mac": "4c:c6:4c:55:ef:66"
  }' | python3 -m json.tool
```

## 2\. 指定 pcap 文件名

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/ptz/capture_at_best" \
  -H "Content-Type: application/json" \
  -d '{
    "mac": "4c:c6:4c:55:ef:66",
    "pcap_filename": "target_device.pcap"
  }' | python3 -m json.tool
```

## 3\. 响应说明

成功时响应示例：

```
{
  "code": 0,
  "data": {
    "queued": true,
    "message": "已发送移动+抓包命令，云台正在移动到最强点",
    "best_position": {"pan": 35.0, "tilt": 10.0},
    "best_rssi": -52,
    "channel": 157,
    "bandwidth": "HT20",
    "source_scan": "location_scan",
    "pcap_filename": "target_device.pcap",
    "stopped_scans": []
  },
  "msg": "ok"
}
```

`stopped_scans` 字段：若调用时有全面扫描或定位扫描正在运行，会列出被停止的扫描类型，例如 `["full_area_scan"]` 或 `["location_scan"]`。

## 4\. 错误场景验证

无任何扫描数据时调用（应返回 404）：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/ptz/capture_at_best" \
  -H "Content-Type: application/json" \
  -d '{"mac": "aa:bb:cc:dd:ee:ff"}' | python3 -m json.tool
```

## 5\. 抓包结束后操作

查看抓包状态：

```
curl -sS "http://127.0.0.1:5000/api/v1/ptz/status" \
  | python3 -c 'import json,sys; d=json.load(sys.stdin); print(json.dumps(d.get("data",{}).get("capture_at_best"), ensure_ascii=False, indent=2))'
```

停止抓包：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/capture/stop" | python3 -m json.tool
```

保存 pcap 文件：

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/capture/save_pcap" \
  -H "Content-Type: application/json" \
  -d '{
    "channel": 157,
    "bandwidth": "HT20",
    "target_mac": "4c:c6:4c:55:ef:66",
    "filename": "target_device.pcap"
  }' | python3 -m json.tool
```

```

# WiFi 连接尝试测试命令

> 用于验证新增的 WiFi 连接尝试功能：将定向网卡临时切换为 managed 模式，尝试连接热点并获取 IP，成功或失败后自动恢复 monitor 模式。

## 1\. 前置条件确认

确认依赖工具可用（wpa\_supplicant / wpa\_cli / dhclient）：

```
which wpa_supplicant wpa_cli dhclient
```

三个都有输出才能正常工作。如果 dhclient 不存在可以用 udhcpc 替代（在 config.json 的 `WiFi连接.DHCP客户端` 配置为 `udhcpc`）。

## 2\. 启动后端并保存完整日志

```
sudo env PYTHONUNBUFFERED=1 /home/ultiwill/ptz_capture_system/ptz_env/bin/python -u run_manager.py 2>&1 \
  | tee wifi_connect_runtime.log
```

## 3\. 发起 WiFi 连接尝试

**基础用法（WPA2 密码连接）：**

把 `你的SSID` 和 `你的密码` 替换成实际值。

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/wifi/connect" \
  -H "Content-Type: application/json" \
  -d '{
    "ssid": "QWQ",
    "password": "ultiwill2025",
    "timeout": 60
  }' | python3 -m json.tool
```

**指定 BSSID（精确连接某台 AP）：**

```
curl -sS -X POST "http://127.0.0.1:5000/api/v1/wifi/connect" \
  -H "Content-Type: application/json" \

## 4\. 查看连接状态

**方式一：从主状态接口查看（推荐）：**

```
curl -sS "http://127.0.0.1:5000/api/v1/ptz/status" \
  | python3 -c 'import json,sys; d=json.load(sys.stdin); print(json.dumps(d.get("data",{}).get("wifi_connect"), ensure_ascii=False, indent=2))'
```

**方式二：专用状态接口：**

```
curl -sS "http://127.0.0.1:5000/api/v1/wifi/connect/status" | python3 -m json.tool
```

`**state**` **字段含义：**

```
queued            → 命令已入队，等待 worker 处理
stopping_others   → 正在等待其他扫描/抓包任务停止（最长 30 秒）
switching_to_managed → 正在将网卡切换为 managed 模式
connecting        → 正在运行 wpa_supplicant 尝试连接
restoring_monitor → 连接结束，正在恢复网卡到 monitor 模式

success           → 连接成功，已获取到 IP 地址（终态）
failed            → 连接失败（密码错误/超时/DHCP失败等）（终态）
cancelled         → 用户手动取消（终态）
timeout           → 连接超时（终态）
error             → 内部错误（终态）
```

**连接中实时轮询（每 2 秒刷新一次，Ctrl+C 停止）：**

```
watch -n 2 'curl -sS "http://127.0.0.1:5000/api/v1/wifi/connect/status" | python3 -m json.tool'
```

**成功时的状态示例：**

```
{
  "state": "success",
  "active": false,
  "terminal": true,
  "connect_id": "wificonn_1749600000000",
  "ssid": "你的SSID",
  "ip_address": "192.168.1.100",
  "elapsed_seconds": 12.3
}
```

**失败时的状态示例：**

```
{
  "state": "failed",
  "active": false,
  "terminal": true,
  "reason": "wrong_password",
  "elapsed_seconds": 15.1
}
```

`reason` 常见值：

```
wrong_password    → 密码错误（4WAY_HANDSHAKE 超时判断）
network_not_found → 找不到目标 SSID（持续 SCANNING 超时判断）
auth_rejected     → 认证被拒
dhcp_timeout      → WPA 连接成功但 DHCP 获取 IP 失败
connect_timeout   → 整体超时（timeout 参数秒数内未完成）
```

## 7\. 验证网卡恢复正常

连接结束（无论成功/失败/取消）后，验证网卡是否已恢复 monitor 模式：

```
iw dev
```

应该能看到定向网卡（默认 `wlan0`）回到 `type monitor`。

如果恢复失败（极端异常情况），手动恢复：

```
sudo ip link set wlan0 down
sudo iw wlan0 set type monitor
sudo ip link set wlan0 up
sudo iw wlan0 set channel 36 HT20
```

tail -f wifi_connect_runtime.log \

  | grep --line-buffered -E 'state=|connect_result=|wpa_state=|connect_id='
