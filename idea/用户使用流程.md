# 用户使用流程 - WiFi信号定位设备

> **文档定位**：以用户视角描述完整的操作路径，供前端交互设计和后端接口联调参考。
> **更新规则**：操作流程变更时同步更新本文档。

---

## 一、设备启动（无需用户操作）

```
设备开机
 └─► 程序自动启动
      ├─► 云台执行归零 / 初始化动作
      ├─► 摄像头开启，开始推流
      └─► 网卡初始化（Monitor 模式配置、全向天线初始化）

初始化完成 → 前端页面可正常使用
```

---

## 二、进入前端主页

用户打开前端页面，看到主页：

- **摄像头实时画面**：展示当前设备朝向，画面中心有十字准心（代表天线对准位置）
- **云台角度**：当前 pan / tilt 数值
- **云台控制面板**：可手动移动云台
- **功能入口**：选择进入"定位"或"描绘"

**用户此时可以：**

- 通过摄像头画面 + 准心确认设备当前对准哪里
- 手动移动云台，画面实时更新

---

## 三、定位功能流程

进入定位功能后，用户可选择以下两个子功能：

---

### 3.1 单设备定位

**适用场景**：已知一个目标设备的 MAC 地址，找到它在扫描区域内信号最强的方向。

**步骤 1：选择扫描区域**

1. 移动云台，将准心对准目标区域**左上角** → 点击确认
2. 移动云台，将准心对准目标区域**右下角** → 点击确认

> ⚡ **选区确认后立即触发**：摄像头在后台开始拍照并拼接全景图，与后续配置**并行进行**。
> 最佳体验：用户配置完成时全景图已就绪，结果可直接叠加显示。
> 若配置速度快于拼接，则等待拼接完成后再启动信号扫描。

**步骤 2：输入目标设备信息**

- MAC 地址（必填）
- 信道（可选）：不填则后端自动遍历全部信道探测，耗时较长
- 带宽（可选）：不填时联动信道自动探测
- 保存采集数据（可选）：选择时自动保存PCAP文件，可供下载

> 提示：知道设备所在信道的情况下填写，可大幅缩短扫描时间。

**步骤 3：启动扫描**

- 云台在选定区域逐点移动，同步采集 WiFi 信号
- 页面展示扫描进度（当前点位 / 总点位）

**步骤 4：查看结果**

- 全景图作为底图展示
- 目标设备在各点位以点标注显示
- **信号最强点高亮标注**（算法定位结果）
- 点击任意标注点 → 查看该点位详细信号数据

---

### 3.2 多设备定位

**适用场景**：同时定位多个目标设备，分别找到每个设备信号最强的方向。

**步骤 1：选择扫描区域**

1. 移动云台，将准心对准目标区域**左上角** → 点击确认
2. 移动云台，将准心对准目标区域**右下角** → 点击确认

> ⚡ 选区确认后同样立即触发全景图拍摄，后台并行进行。

**步骤 2：添加多个目标设备**

- 依次添加多个 MAC 地址
- 每个设备可单独配置信道/带宽（可选），不填则自动探测

**步骤 3：启动扫描**

- 云台在区域内逐点移动，对所有目标设备同步采集信号
- 页面展示扫描进度

**步骤 4：查看结果**

- 全景图作为底图
- 每个目标设备均有各自的信号最强方向标注
- 多设备结果展示方式（列表切换 / 同屏多标注等）**待与前端确认**
- 点击某设备的标注点 → 查看该设备在该点位的详细信号数据

## 四、描绘功能流程

### 适用场景

用户想了解指定区域内有哪些 WiFi 设备，以及各设备的信号分布情况。

### 操作步骤

**步骤 1：选择扫描区域**

描绘支持以下几种区域选择方式：


| 选择方式          | 操作                  | 说明                              |
| ------------- | ------------------- | ------------------------------- |
| **对角框选（单区域）** | 移动云台对准左上角确认 + 右下角确认 | 最常用，框选一个矩形区域                    |
| **多小区域**      | 重复框选操作，添加多个不连续区域    | 每个小区域独立扩展后依次扫描                  |
| **全范围扫描**     | 不选区域，直接使用全范围        | 扫描云台完整可动范围（举例：水平 180° × 垂直 40°） |


> ⚠️ **扫描范围自动扩展**：无论选哪种方式，实际扫描范围均会在用户选区基础上向外静默扩展一圈，防止边界格子因旁瓣信号产生误判。扩展幅度不对用户显示，属于后台保护机制。

> ⚡ **选区确认后立即触发**：摄像头在后台开始拍照并拼接全景图，与用户后续配置并行进行。

**步骤 2：选择附加选项（可选）**

- **客户端扫描开关**：开启后重点扫描 STA 客户端及其关联 AP
- **星链识别**：默认开启，无需手动操作

**步骤 3：启动扫描**

- 云台在区域内逐点移动，全面采集 WiFi 信号
- 页面展示扫描进度

**步骤 4：查看结果**

- 全景图作为底图
- 区域内所有探测到的设备以点标注叠加在全景图上
- 点击标注点 → 查看该位置设备详细信息
- 星链设备特别标注

### 描绘结果的时间轴（后续功能）

- 描绘支持定时重复扫描，每轮结果自动保存
- 后续将支持时间轴拖动，查看不同时刻的设备分布变化

## 五、操作注意事项


| 注意点            | 说明                                                                             |
| -------------- | ------------------------------------------------------------------------------ |
| 同一时间只能运行一个扫描任务 | 定位和描绘不能同时进行，需停止当前任务再开启新任务                                                      |
| 信道留空会增加耗时      | 定位时不填信道，后端需遍历全部信道，扫描时间明显增长                                                     |
| 准心代表天线方向       | 准心不是摄像头光轴，是经过安装偏差校准后的天线对准位置                                                    |
| **与目标保持有效距离**  | 摄像头与天线之间存在固定高度差，距离目标过近时偏差不可忽略，图像映射与定位结果会不准确。本设备适合对**有一定距离的目标**进行方向定位，不适合极近距离使用 |
| 扫描中可随时停止       | 停止后已完成点位的数据会保留，不会丢失                                                            |
