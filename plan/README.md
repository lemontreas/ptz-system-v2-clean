## 维护规则

- 默认不要为一次性小改动创建 `plan/` 优化计划。
- 用户明确要求“做优化计划 / 记录到 plan / 这是长期优化”，或任务明显跨多步、跨文件、跨会话时，才创建或更新计划。
- 每个优化计划最多 10 个 Todo；准备加入第 11 个优化点时，必须按主题新建计划并更新本索引，避免单计划过长增加阅读和上下文成本。
- 新建 `plan/*.md` 后，必须在“当前计划”表格中新增一行。
- 计划状态变化时，必须使用“固定状态”中的枚举，并同步更新计划正文状态。
- 每次完成关键 Todo 或调整下一步时，必须同步更新“最后更新”和“下一步 / 备注”。
- “最后更新”统一使用 `YYYY-MM-DD HH:mm`，默认时区 `Asia/Shanghai`。
- 如果计划被 `question/` 问题阻塞或验证，必须在“下一步 / 备注”中简要标注相关问题路径。
- 历史计划条目不强制一次性补全；后续触碰、状态变化或新增记录时，必须顺手补齐到当前格式。
- 修改本索引或计划模板后，不要求运行代码测试，但必须按 `agent.md` 的“文档自检”清单检查模板、索引、状态枚举、时间格式和触发条件。
- 模板文件不加入“当前计划”表格，只放在“模板”区域。
