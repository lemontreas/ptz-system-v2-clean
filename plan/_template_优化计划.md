# 简短主题优化计划

> 文件命名建议：`YYYY-MM-DD_简短主题_优化计划.md`
> 用途：记录持续优化的 Todo 总览、每项详细方案、验证结果和后续追加项。
> 固定状态：`待整理` / `待评审` / `待执行` / `进行中` / `待验证` / `已完成` / `暂缓` / `取消`
> 创建条件：用户明确要求，或任务明显需要跨多步、跨文件、跨会话推进。
> 时间格式：`YYYY-MM-DD HH:mm`，默认时区 `Asia/Shanghai`
> 容量限制：每个优化计划最多 10 个 Todo；第 11 个优化点必须新建计划并更新 `plan/README.md`。

## 维护规则

- 默认不要为一次性小改动创建优化计划。
- 完成 Todo 后，把 `- [ ]` 改为 `- [x]`，并补充对应详细说明里的进度记录。
- 所有进度记录时间统一使用 `YYYY-MM-DD HH:mm`，默认时区 `Asia/Shanghai`。
- 新增优化点时，追加 Todo 和对应详细说明，不要只写在正文里。
- 每个计划最多 10 个 Todo；超过上限时新建主题明确的优化计划，避免文档过长增加阅读和上下文成本。
- 暂缓或取消的 Todo 保留原条目，并写明原因。
- 若用户确认把阻塞性问题记录到 `question/`，必须在本计划的“关联问题”和对应 Todo 进度记录中反向引用。
- 关联问题解决后，必须回到对应 Todo，补充最终结论和下一步状态。
