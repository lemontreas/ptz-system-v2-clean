## 维护规则

- 默认不要自动创建 `question/` 问题记录；只有用户明确标记或确认后才创建。
- 新建 `question/*.md` 后，必须在“当前问题”表格中新增一行。
- 问题状态变化时，必须使用“固定状态”中的枚举，并同步更新文件名、正文状态和本索引状态。
- 每次新增关键排查结论时，必须同步更新“最后更新”和“当前卡点 / 结论”。
- “最后更新”统一使用 `YYYY-MM-DD HH:mm`，默认时区 `Asia/Shanghai`。
- 问题解决后，必须把状态改为 `已解决`，并在“当前卡点 / 结论”中写清根因或最终结论摘要。
- 如果问题关联某个 `plan/` 计划，必须在“当前卡点 / 结论”中简要标注相关计划路径。
- 历史问题条目不强制一次性补全；后续触碰、状态变化或新增记录时，必须顺手补齐到当前格式。
- 修改本索引或问题模板后，不要求运行代码测试，但必须按 `agent.md` 的“文档自检”清单检查模板、索引、状态枚举、时间格式和触发条件。
- 模板文件不加入“当前问题”表格，只放在“模板”区域。
